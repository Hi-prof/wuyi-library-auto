# Android 手测脚本

> 可选验证项，不阻塞合入；用于覆盖需要真机、系统权限或真实网络环境的功能。

## 前置条件

- Android 真机一台，已安装当前调试包。
- 已登录至少一个有效账号，并保存可用座位 / 预约任务。
- 系统蓝牙、定位、网络权限按测试场景开启或关闭。

## 1. BLE 扫描命中

1. 打开蓝牙和定位权限。
2. 进入会触发签到的任务或调试路径。
3. 保证附近存在目标 iBeacon minor。
4. 触发签到。
5. 期望结果：
   - 8 秒内完成扫描。
   - 命中后停止扫描。
   - 签到审计记录包含 `matchedMinor`、`scanDurationMillis` 和成功状态。

## 2. BLE 扫描未命中

1. 打开蓝牙和定位权限。
2. 保证附近不存在任务期望的 iBeacon minor。
3. 触发签到。
4. 期望结果：
   - 扫描到超时后停止。
   - 不发起伪成功签到。
   - 审计记录显示未命中或超时原因。

## 3. 批量签到

1. 准备两个及以上账号，其中至少一个账号可签到。
2. 进入账号管理页。
3. 点击“全部签到”。
4. 立刻再次点击“全部签到”。
5. 对失败账号点击单条“重试”。
6. 期望结果：
   - 首次点击 100 ms 内返回运行态。
   - 第二次点击进入冷却态，不重复派发。
   - 单条重试只刷新对应账号行。
   - 执行日志摘要以“批量签到”开头。

## 4. 设置中心展示

1. 启动应用，确认底部有“首页 / 账号 / 预约 / 任务 / 设置”5 个 tab。
2. 进入设置中心。
3. 检查常用、监控、同步、诊断四组入口。
4. 检查签到监控、看门狗状态、权限、运行说明、自动预约说明、诊断日志、服务端同步和构建信息入口。
5. 分别进入页面。
6. 期望结果：
   - 入口都可见且可导航。
   - 设置中心不再展示已移除的 Wi-Fi 重连、校园网认证、网络监控入口。
   - 签到监控展示最近签到 / BLE 审计与 24h 错误聚合。
   - 看门狗状态展示心跳、失败计数、关键 unique work 和下次预计执行时间。
